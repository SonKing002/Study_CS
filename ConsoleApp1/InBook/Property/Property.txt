
필드의 맴버변수 원본 그대로 public 사용해도 되는 것을 왜 보호처리할까

		"철수와 영희"
	
		철수 :  public int 돈 50_000;
		영희 : "철수 돈 내꺼" (멋대로 가져감) == 데이터 변조 위험이 있다.

		직접 접근하는 일이 있으면 안된다.
		영희 : "철수야, 돈좀 줄 수 있을까?" 

		데이터를 읽고 싶을때 요청,
		철수는 데이터를 원하는 만큼 전달


은닉성, 편의성 : 프로퍼티 사용
ex.
	인스펙터 노출되는 것들은 원본이 아니라, 맴버 프로퍼티
	애니메이션 동작 : Properties 추가

	수정되면 오작동을 하기 때문


class MyClass
{
	private int myField;
	

//1. Get/Set 메서드 만들어서 제어

	public int GetMyField() 
	{
		return myField;
	}

	public void SetMyField(int value)
	{
		myField = value
	}


//2. C# 제공 

	public int MyField
	{
		get
		{
			return myField;
		}
		set
		{
			myField = value;
		}
	}

	//2_1. 일부만 사용 가능
	public int MyField
	{
		get
		{
			return myField;
		}
	}//읽기 전용 

}
